Infraestructura de Streaming Pirata — Investigación

Compilado para análisis forense y desmantelamiento de sitios ilegales. Fuentes: TorrentFreak, MPA Notorious Markets, Cloudflare Transparency Reports, ACE, Verimatrix, análisis propio.

1. Piracy-as-a-Service (PaaS) — Modelo actual dominante

Sitios como Sflix, Myflixerz, HDtoday no hostean ni suben videos. Usan un backend compartido:

Frontend (Sflix, Myflixerz, HDtoday, etc.)
  └── Cloudflare (CDN/escudo, oculta IP real)
       └── Backend PaaS (MegaCloud, VidCloud, RapidCloud)
            └── Servidores físicos (Ucrania / Rusia / Países Bajos)
                 └── Videos almacenados + transcodificación

Proveedores PaaS conocidos

Servicio Antecesor Estado Notas
MegaCloud — Activo Backend principal de la red Sflix/Myflixerz
VidCloud — Activo Alternativa a MegaCloud
RapidCloud — Activo Similar a los anteriores
2embed.to — Tumbado Jun 2023 por ACE El original, resucitó como MegaCloud/VidCloud

Impacto: Si el PaaS cae, caen decenas de sitios

  • Abril 2026: Sflix, Myflixerz, HDtoday, Moviesjoy, FlixHQ, ~20 dominios más cayeron simultáneamente con error Cloudflare 521
  • La causa fue la caída del backend PaaS compartido
  • Ningún grupo antipiratería reclamó el crédito
  • La red Sflix/Myflixerz sola generaba 622 millones de visitas/mes (MPA, Ago 2025)

Fuente: TorrentFreak — “Sflix, Myflixerz, HDtoday, and other Pirate Sites Go Dark as Backend Infrastructure Fails” (23 Abr 2026)

Citación textual de la MPA (Notorious Markets 2025)

“These sites rely on their own PaaS infrastructure (formerly known as 2embed[.]to, which ACE took down in June 2023) and despite enforcement, they continue to thrive through alternative domains and backend hosting on platforms such as MegaCloud, VidCloud, and RapidCloud. Unlike the previous CMS model, which explicitly enabled pirate sites to embed movies and monetize streams, this new model functions as a backend hosting network powering popular pirate domains such as those mentioned above. These services act as a media source server, serving video files directly allowing a myriad of sites to provide streams to users.”

2. CDN Leeching — TV en vivo pirata

Los sitios de TV en vivo no hostean nada. Le hacen “leech” a CDNs legítimos:

Canal oficial (DAZN, ESPN, DirecTV)
  → Akamai CDN (300k+ servidores en 4000 ubicaciones)
    → Sitio pirata extrae URL .mpd + DRM ClearKey con DevTools
      → Pone JWPlayer / Clappr en GitHub Pages o Cloudflare Pages
        → El video viaja DIRECTAMENTE de Akamai al usuario final
          → El sitio pirata NO paga ancho de banda

Por qué funciona

  • Akamai no puede distinguir entre un usuario legítimo y uno que copió la URL del manifiesto
  • Las URLs y keys ClearKey van en texto plano en el JavaScript del player
  • Akamai factura al dueño legítimo del canal por el tráfico, incluyendo el que generan los piratas

Métodos de extracción

  1. Abrir página oficial del canal (DAZN, ESPN, etc.)
  2. Chrome DevTools → Network → filtrar por “mpd” o “m3u8”
  3. Capturar URL del manifiesto (Akamai, Cloudfront)
  4. Filtrar por “license” o “key” → capturar keyId + key
  5. Probar en JWPlayer/Clappr localmente

3. ClearKey DRM — El talón de Aquiles

ClearKey es un sistema DRM deliberadamente débil definido por el W3C:

Característica ClearKey Widevine L1 Widevine L3
Módulo de descifrado Ninguno (texto plano) CDM hardware (TEE) CDM software
Keys visibles en JS Sí No No (pero extractable)
Seguridad Nula Alta Media
Costo para el canal Gratis Licencia costosa Licencia

Herramientas para descargar streams con ClearKey

# dash-mpd-cli
dash-mpd-cli --key eb676abbcb345e96bbcf616630f1a3da:100b6c20940f779a4589152b57d2dacb https://cdn.ejemplo.com/stream.mpd

# yt-dlp (soporta algunos DASH con --allow-unplayable-formats)
yt-dlp --allow-unplayable-formats https://ejemplo.com/stream.mpd

Por qué los canales usan ClearKey si es inseguro

  • Para TV en vivo, la ventana de explotación es corta (horas)
  • Las URLs y keys expiran regularmente
  • ClearKey es gratis; Widevine requiere licencias caras
  • El overhead técnico es mínimo
  • El deporte en vivo se consume en el momento — no importa si alguien captura la key

4. Hydra Sites — Concepto de la MPA

La MPA acuñó el término “Hydra Sites” para describir redes organizadas de piratería:

Características

  • Operados por redes criminales organizadas (no individuos)
  • Infraestructura PaaS compartida
  • Docenas de dominios que parecen sitios independientes pero apuntan al mismo backend
  • Si cortás una “cabeza” (dominio), crecen dos más (dominios de respaldo)
  • Modelo de negocio con monetización agresiva (popunders, CPA, cripto)

Cloudflare

  • Potencia ~20% de los sitios web, incluyendo mucha piratería
  • Reporte de transparencia H1 2025:
    • 124,872 reportes de copyright recibidos
    • 54,357 acciones de takedown (3,800% vs H2 2024)

    • Empezaron API para derechohabientes → takedowns automatizados
    • Bloqueo voluntario de sitios pirata en UK (respuesta HTTP 451)
    • Siguen rechazando bloqueo a nivel DNS (1.1.1.1)
  • Demandado en Japón: ordenado a pagar 500M yenes por facilitar piratería (Nov 2025)

Fuente: Cloudflare Transparency Report H1 2025; TorrentFreak, Dic 2025

ACE (Alliance for Creativity and Entertainment)

Caso Calcio (Sep 2025):

  • Mayor sitio de deportes italiano: 123M visitas/año
  • 80% del tráfico de Italia (Serie A)
  • Operaba desde Moldova — país que tradicionalmente ignora DMCA
  • Tenía 134 dominios para evadir bloqueos (calcio.ws, .la, .autos, .baby, .beauty, etc.)
  • ACE + DAZN lograron el cierre sin acción legal directa
  • Contexto: Moldova busca entrar a la UE → necesita alinear leyes de propiedad intelectual
  • El operador “accedió a cerrar después de ser contactado” (presión diplomática)

Caso Streameast (Sep 2025):

  • ACE anunció el cierre de la red Streameast
  • El original sobrevivió — solo cayeron algunos clones
  • Intervención del DHS (Homeland Security) con secuestro de dominios

Fuente: TorrentFreak — “ACE & DAZN Shut Down a Major Sports Piracy Site in a DMCA Ignored Country” (10 Sep 2025)

LaLiga (España)

  • Usa bloqueo por IP (no por dominio)
  • Cloudflare criticó públicamente el enfoque como “chapucero” (clunky)
  • El bloqueo por IP afecta sitios legítimos que comparten IPs en Cloudflare
  • Cloudflare: “LaLiga ha sugerido que sus intereses comerciales deberían primar sobre el derecho de los usuarios españoles a acceder a Internet”

1. Modelo VOD (Video on Demand)

Cómo funciona

Es un modelo mucho más simple que el de TV en vivo:

  1. Alguien sube el video a un hosting gratuito como StreamTape, Vidoza, VOE.sx
    • Estos sitios son como YouTube pero sin control de copyright
  2. El uploader obtiene un embed code (un iframe)
  3. El sitio pega ese iframe rodeado de publicidad
  4. WordPress + SEO para atraer tráfico de Google
Alguien sube video → StreamTape genera iframe → Sitio pega iframe + ads → Visitante ve video

Hostings de video populares

Servicio Embed Ancho de banda Paga al uploader Propietario
StreamTape iframe Ilimitado $5–15/10k views Desconocido (Ucrania/Rusia)
Vidoza iframe Ilimitado $4–12/10k views Desconocido
VOE.sx iframe Ilimitado $3–10/10k views Desconocido
MixDrop iframe Ilimitado Sí Desconocido
Streamruby iframe Ilimitado Sí Desconocido

Estos hostings no pagan por el tráfico — al revés, ganan dinero con los ads que ponen ellos mismos en la página del video. El uploader recibe una comisión.

Por qué usan estos hostings en vez de YouTube

  • YouTube detecta y elimina contenido con copyright en minutos
  • StreamTape/Vidoza ignoran reclamos DMCA
  • YouTube comprime el video; estos hostings mantienen la calidad original
  • YouTube no permite cierto contenido (adulto, violento); estos sí

</iframe>

2. Análisis de internetchicks.com

Estructura del sitio

internetchicks.com
├── WordPress (tema personalizado)
├── Artículos con estructura:
│   ├── Título SEO: "Top 10 OnlyFans Videos 2025"
│   ├── Banner publicitario (AdSense o similar)
│   ├── Video embebido (StreamTape / Vidoza / PlayMogo / VOE)
│   ├── Botones de afiliados a cam sites (Stripchat, Chaturbate, CAM4)
│   ├── Más banners
│   └── Popup scripts (al hacer clic en cualquier parte)
└── Plugins comunes:
    ├── SEO (Rank Math / Yoast)
    ├── Cache (WP Rocket / W3 Total)
    ├── Anti-adblock (varios)
    └── Popup maker

Cómo consiguen los videos

  1. Automatizado: Bots de Telegram rastrean canales de OnlyFans/Fansly
  2. Manual: Alguien compra contenido y lo re-subre a los hostings
  3. Comunidad: Usuarios envían enlaces a cambio de premium
  4. Re-upload: Bajan de otros sitios pirata y re-suben a StreamTape

Selectores de video (múltiples fuentes)

  <button onclick="loadVideo('streamtape', 'Oo49Gep071SZyqg')">StreamTape #1</button>
  <button onclick="loadVideo('vidoza', 'abc123')">Vidoza</button>
  <button onclick="loadVideo('voe', 'def456')">VOE</button>
</div>
<div id="video-container">
  <iframe id="player" src="about:blank"></iframe>

Por qué múltiples fuentes

  • Si StreamTape elimina el video, aún funciona en Vidoza
  • Distribuyen el ancho de banda entre varios hosts
  • Si un host está caído, el usuario puede cambiar
  • Diferentes hosts tienen diferentes políticas de retención

3. Diferencia clave: VOD vs TV en vivo

Característica Video on-demand (internetchicks) TV en vivo (COLFUT)
Contenido Video ya grabado Stream en vivo
Host del video StreamTape/Vidoza (gratis) CDN del canal (Akamai, Cloudfront)
Embed Solo pegar iframe Necesita DASH + DRM keys
Complejidad técnica Baja (WordPress + iframe) Alta (DASH, ClearKey, players)
Caducidad Permanente (hasta que lo borren) Las URLs expiran en horas/días
SEO Alto (Google indexa cada video) Bajo (el link muere cuando acaba el evento)
Mantenimiento Subir archivos periódicamente Actualizar streams constantemente
Plataforma WordPress HTML estático (GitHub Pages)

VOD — flujo completo

Creador de contenido (OF/Fansly/etc.)
  → Bot descarga el video
    → Sube a StreamTape
      → Obtiene iframe
        → WordPress publica artículo con iframe + ads
          → Google indexa
            → Usuario busca "video X gratis"
              → Hace clic → ve anuncios → ve video

TV en vivo — flujo completo

Canal oficial (DAZN/ESPN)
  → Emite stream en vivo vía Akamai CDN
    → Sitio pirata extrae URL .mpd + DRM keys con DevTools
      → Crea HTML con JWPlayer/Clappr + keys
        → Sube a GitHub Pages
          → Usuario visita la URL
            → HTML carga player → player descifra stream con keys → usuario ve TV

Por qué TV en vivo es más riesgoso

  • Los canales (DAZN, ESPN, DirecTV) tienen equipos legales que persiguen
  • Las URLs de stream expiran rápido — hay que actualizarlas cada horas
  • Los CDN pueden bloquear por IP o Referer
  • DRM ClearKey puede rotar la clave
  • Los dominios de GitHub Pages son reportados y bloqueados por ISP

4. Economía del streaming pirata

Fuentes de ingreso

Popunders / Popups (la principal fuente)

Redes como PopAds y PropellerAds pagan por cada impresión de popup:

Red CPM típico Pago por 100k visitas
PopAds $0.50–$3.00 $50–$300
PropellerAds $1.00–$5.00 $100–$500
Adsterra $0.50–$2.00 $50–$200
  • Los popunders se abren en cada clic dentro del sitio
  • Un usuario puede generar 3–10 popups por visita
  • Los popups muestran: casinos, apuestas, cam sites, malware fingido

Google AdSense

  • Paga $0.10–$0.50 CPM (mucho menos que popups)
  • Muchos sitios no pueden usarlo porque Google lo prohibe por copyright
  • Los que sí lo usan tienen cuidado de no poner contenido infractor en la página principal

CPA (Cost Per Action) — el verdadero dinero

Ofertas de afiliados que pagan por registro o acción:

Tipo de oferta Pago por acción Ejemplo
Cam sites (Stripchat, Chaturbate, CAM4) $10–$100 por registro Usuario se registra gratis
Casinos online $50–$300 por depósito Usuario deposita $20
Encuestas $1–$5 por completar Usuario responde 5 preguntas
Instaladores PUP (adware) $1–$5 por instalación Usuario instala “optimizador”
Webrtc leaks / VPNs $3–$10 por venta Usuario compra VPN

Ejemplo real: internetchicks.com (estimación)

  • Tráfico SimilarWeb: ~500k–1M visitas/mes
  • Fuentes de tráfico: 60% búsqueda orgánica, 25% directo, 10% redes sociales, 5% referidos
  • Ingreso estimado: $15k–$40k/mes

  • Dueño: probablemente 1 persona con WordPress + 50–100 artículos

Economía de TV en vivo vs VOD

TV en vivo tiene menos visitas pero CPM más alto:

  • Un partido importante genera picos de tráfico: 50k–200k viewers simultáneos
  • Los popunders en eventos en vivo convierten mejor
  • Más riesgo legal (los canales persiguen más agresivamente)
  • Las URLs expiran — hay que actualizar constantemente
  • Ingreso típico por evento grande: $500–$5,000 en un día

Quién gana realmente

  • El hosting del video y el sitio web están separados: StreamTape solo provee el iframe
  • Si cierran un servidor, tienen 10 más
  • No aceptan reclamos de copyright de manera significativa

6. Infraestructura anti-bloqueos

Cómo evitan ser derribados

Capa 1: Cloudflare

Usuario → Cloudflare (oculta IP real) → Servidor real
           ┌─────────────┐
           │ CDN + proxy │
           │ DDoS protect│
           │ IP oculta   │
           └─────────────┘
  • Cloudflare actúa como escudo: el servidor real nunca es visible
  • El sitio se vuelve resistente a ataques DDoS
  • Si Cloudflare recibe una orden DMCA, entregan los datos del dueño de la cuenta (por eso usan datos falsos)

Capa 2: Dominios dinámicos

  • Tienen decenas de dominios registrados
  • Cuando uno es bloqueado por el ISP, rotan a otro
  • Usan TLDs difíciles de bloquear: .com, .net, .org, .cc, .io, .tv
  • Algunos usan dominios gratis (.tk, .ml, .ga — Freenom) que son imposibles de rastrear

Capa 3: Hosting del contenido

Contenido ilegal (video)
  → Hosteado en StreamTape (Ucrania/Rusia)
    → Fuera de alcance DMCA de EE.UU./Europa
      → No responden a takedown notices

Capa 4: Múltiples mirrors

Cada sitio tiene 3–10 copias en diferentes dominios:

  • sitioweb.com
  • sitioweb.net
  • sitioweb.org
  • sitioweb.cc
  • ver-sitioweb.com
  • sitioweb-online.com

Distribuyen el enlace en Telegram/Redes: “Si no carga, usa este mirror”

Capa 5: Sin servidores propios

  • GitHub Pages (gratis, tráfico ilimitado)
  • Netlify (gratis, 100GB)
  • Cloudflare Pages (gratis, ilimitado)
  • Vercel (gratis)

El sitio es estático — solo HTML, CSS y JS. Sin backend. Sin base de datos. Sin servidor que derribar.

El dinero

  • Payoneer: tarjetas virtuales, cuentas en múltiples países
  • Criptomonedas: Bitcoin, Monero (anónimo)
  • Cuentas bancarias en países sin cooperación internacional

2. Cómo capturar URLs y DRM keys con DevTools

Este es el proceso que usan para conseguir los streams:

Paso 1: Abrir la página oficial del canal

  • Ve a la web/app oficial de DAZN, ESPN, etc.
  • Abre Chrome/Edge DevTools: F12 o Click derecho → Inspeccionar

Paso 2: Ir a la pestaña Network

  • Haz clic en la pestaña Network
  • Filtra por “media” o “mpd” o “m3u8”
  • Inicia la reproducción del video

Paso 3: Capturar el stream

  • Aparecerá una petición a un CDN (Akamai, Cloudfront, etc.)
  • El URL será algo como:
    • https://xxxx.akamaized.net/dashdrm/dazn-linear-016/stream.mpd
    • https://xxxx.cloudfront.net/something/manifest.mpd

Paso 4: Capturar las DRM keys

  • En la misma pestaña Network, filtra por “license” o “key”
  • Las peticiones de licencia DRM devuelven las keys
  • En las Response Headers o Response verás algo como: 
    {
  "keyId": "82f8e2a17dac44c0a18f660479349c59",
  "key": "1b144f73e6fefe91cd05f850e2b589d0"
}

Paso 5: Probar el stream localmente

Crea un HTML con el player y pega el URL + keys. Si funciona, ya tienes el embed.

Notas importantes:

  • Las URLs y keys expiran (tienen tiempo de vida)
  • Algunos CDN verifican el Referer o User-Agent
  • Las keys pueden cambiar cada sesión
  • Los sitios pirata actualizan sus archivos manualmente o con bots

8. Preguntas frecuentes

¿De dónde sacan las URLs y DRM keys?

De la página/app oficial del canal usando Chrome DevTools (Network → filtrar por “mpd” o “license”). Las keys vienen en la respuesta de la licencia DRM.

¿Cada cuánto cambian las URLs?

Depende del canal. Algunas duran horas, otras días. Los sitios grandes tienen bots que las actualizan automáticamente.

¿Por qué usan GitHub Pages?

Es gratis, permite tráfico ilimitado, y aloja archivos estáticos. El stream no pasa por GitHub — solo el HTML con el player. El video va directo del CDN al navegador.

¿Cómo ganan dinero?

  • Google AdSense
  • Popups y redirecciones a sitios de apuestas/casinos
  • Links de afiliados
  • Anuncios directos

¿Qué es ClearKey DRM?

Es un sistema de encriptación de video. El stream está encriptado y el navegador necesita una “llave” (key) para descifrarlo. Esa llave va en el JavaScript del player. Sin ella, solo se ve una pantalla negra.

Diferencia entre DASH (.mpd) y HLS (.m3u8)

Característica DASH (.mpd) HLS (.m3u8)
Quién lo usa DAZN, ESPN, DirecTV Canales abiertos, YouTube
DRM Sí (ClearKey, Widevine) Generalmente no
Calidad Alta (hasta 4K) Variable
Complejidad Requiere player especial Solo video tag + hls.js

¿Qué es Akamai?

Es un CDN (Content Delivery Network). DAZN, ESPN y otros grandes usan Akamai para distribuir sus streams. Los URLs de Akamai se ven así:

https://xxxx.akamaized.net/dashdrm/dazn-linear-016/stream.mpd

¿Puedo usar VLC para probar estos streams?

Los streams con DRM ClearKey NO se pueden abrir en VLC directamente porque VLC no soporta ClearKey en DASH. Necesitas un navegador con el player correcto (JWPlayer, Clappr, Shaka Player).

2. Cómo capturar URLs y DRM keys con DevTools

Este es el proceso que usan para conseguir los streams:

Paso 1: Abrir la página oficial del canal

  • Ve a la web/app oficial de DAZN, ESPN, etc.
  • Abre Chrome/Edge DevTools: F12 o Click derecho → Inspeccionar

Paso 2: Ir a la pestaña Network

  • Haz clic en la pestaña Network
  • Filtra por “media” o “mpd” o “m3u8”
  • Inicia la reproducción del video

Paso 3: Capturar el stream

  • Aparecerá una petición a un CDN (Akamai, Cloudfront, etc.)
  • El URL será algo como:

Paso 4: Capturar las DRM keys

  • En la misma pestaña Network, filtra por “license” o “key”
  • Las peticiones de licencia DRM devuelven las keys
  • En las Response Headers o Response verás algo como: 
    {
  "keyId": "82f8e2a17dac44c0a18f660479349c59",
  "key": "1b144f73e6fefe91cd05f850e2b589d0"
}

Paso 5: Probar el stream localmente

Crea un HTML con el player y pega el URL + keys. Si funciona, ya tienes el embed.

Notas importantes:

  • Las URLs y keys expiran (tienen tiempo de vida)
  • Algunos CDN verifican el Referer o User-Agent
  • Las keys pueden cambiar cada sesión
  • Los sitios pirata actualizan sus archivos manualmente o con bots

8. Preguntas frecuentes

¿De dónde sacan las URLs y DRM keys?

De la página/app oficial del canal usando Chrome DevTools (Network → filtrar por “mpd” o “license”). Las keys vienen en la respuesta de la licencia DRM.

¿Cada cuánto cambian las URLs?

Depende del canal. Algunas duran horas, otras días. Los sitios grandes tienen bots que las actualizan automáticamente.

¿Por qué usan GitHub Pages?

Es gratis, permite tráfico ilimitado, y aloja archivos estáticos. El stream no pasa por GitHub — solo el HTML con el player. El video va directo del CDN al navegador.

¿Cómo ganan dinero?

  • Google AdSense
  • Popups y redirecciones a sitios de apuestas/casinos
  • Links de afiliados
  • Anuncios directos

¿Qué es ClearKey DRM?

Es un sistema de encriptación de video. El stream está encriptado y el navegador necesita una “llave” (key) para descifrarlo. Esa llave va en el JavaScript del player. Sin ella, solo se ve una pantalla negra.

Diferencia entre DASH (.mpd) y HLS (.m3u8)

Característica DASH (.mpd) HLS (.m3u8)
Quién lo usa DAZN, ESPN, DirecTV Canales abiertos, YouTube
DRM Sí (ClearKey, Widevine) Generalmente no
Calidad Alta (hasta 4K) Variable
Complejidad Requiere player especial Solo video tag + hls.js

¿Qué es Akamai?

Es un CDN (Content Delivery Network). DAZN, ESPN y otros grandes usan Akamai para distribuir sus streams. Los URLs de Akamai se ven así:

https://xxxx.akamaized.net/dashdrm/dazn-linear-016/stream.mpd

¿Puedo usar VLC para probar estos streams?

Los streams con DRM ClearKey NO se pueden abrir en VLC directamente porque VLC no soporta ClearKey en DASH. Necesitas un navegador con el player correcto (JWPlayer, Clappr, Shaka Player).

Cómo funciona

Es un modelo mucho más simple que el de TV en vivo:

  1. Alguien sube el video a un hosting gratuito como StreamTape, Vidoza, VOE.sx

  2. El uploader obtiene un embed code (un iframe)
  3. El sitio pega ese iframe rodeado de publicidad
