Urbanterror Fix

May 08, 2026
Ai

No puedo jugar a UrbanTerror desde la TTY (abre el juego pero se cierra), asi que le pedi a Claude AI que me lo arregle.

¡Buena aclaración! En Arch el proceso es más limpio. Vamos:

1. Instalar dependencias

sudo pacman -S git base-devel sdl2 mesa libgl \
  libpulse alsa-lib libdrm libgbm \
  curl speex opusfile

Verificá que SDL2 tenga KMSDRM (en Arch viene habilitado por defecto):

sdl2-config --version

# Confirmar backend KMSDRM:
find /usr/lib -name "libSDL2*" | xargs strings | grep -i kms

2. Clonar y compilar ioQuake3

git clone https://github.com/ioquake/ioq3.git
cd ioq3

make \
  USE_SDL=1 \
  USE_OPENGL2=0 \
  USE_CURL=1 \
  USE_VOIP=1 \
  ARCH=x86_64 \
  -j$(nproc)

El binario queda en:

./build/release-linux-x86_64/ioquake3.x86_64

3. Apuntar al contenido de Urban Terror

SDL_VIDEODRIVER=kmsdrm \
./build/release-linux-x86_64/ioquake3.x86_64 \
  +set fs_game q3ut4 \
  +set fs_basepath /home/esotericwarfare/urbanterror/UrbanTerror43

4. Si KMSDRM falla, verificar permisos

# Tu usuario debe estar en el grupo 'video' y 'input'
sudo usermod -aG video,input $USER

# Verificar dispositivos DRM disponibles:
ls -la /dev/dri/

# Debería verse algo como:
# card0, card1, renderD128

Después de agregar al grupo, cerrar sesión y volver a entrar para que tome efecto.

5. Crear script de arranque

Para no escribir todo cada vez:

cat > ~/launch-urt.sh << 'EOF'
#!/bin/bash
SDL_VIDEODRIVER=kmsdrm \
SDL_RENDER_DRIVER=opengles2 \
/home/esotericwarfare/ioq3/build/release-linux-x86_64/ioquake3.x86_64 \
  +set fs_game q3ut4 \
  +set fs_basepath /home/esotericwarfare/urbanterror/UrbanTerror43 \
  "$@"
EOF

chmod +x ~/launch-urt.sh

Posibles problemas en Arch

Problema Solución
KMSDRM not available SDL2 del sistema no tiene el backend, reinstalar con sudo pacman -S sdl2
could not open display Normal en TTY, KMSDRM no necesita display
signal 11 igual Probar SDL_VIDEODRIVER=offscreen para descartar el problema
Pantalla negra Agregar +r_mode -2 +r_fullscreen 1 al comando
Sin sonido Agregar SDL_AUDIODRIVER=alsa o =pulseaudio

¿Querés que también vea cómo hacer que arranque automáticamente desde TTY1 al iniciar el sistema?

El problema es claro: Urban Terror está usando un binario precompilado de 2018 que intenta inicializar OpenGL vía SDL2 con un display X11/Wayland, y cuando no encuentra ninguno desde la TTY, recibe un Segmentation Fault (signal 11).

Los juegos que mencionás (Doom via GZDoom/Chocolate, SuperTuxKart, 0ad) funcionan en TTY porque usan KMS/DRM directamente o tienen backends que no dependen de X11.

Urban Terror 4.3 es un binario cerrado basado en ioQuake3, así que no podés recompilarlo fácilmente. Tus opciones son:

Opción 1: Usar Xvfb + x11vnc (sin monitor, todo virtual)

No es TTY pura, pero funciona sin GPU display:

Xvfb :1 -screen 0 1280x720x24 &
DISPLAY=:1 ./Quake3-UrT.x86_64

Opción 2: Forzar SDL2 con backend DRM/KMSDRM

SDL2 tiene un backend KMSDRM que va directo al framebuffer sin X11. El binario de UrT usa SDL2, así que podés intentar forzarlo:

SDL_VIDEODRIVER=kmsdrm ./Quake3-UrT.x86_64

Si falla, verificá que tu usuario tenga acceso a los dispositivos DRM:

ls -la /dev/dri/
# Tu usuario debe estar en el grupo 'video' o 'render'
sudo usermod -aG video,render $USER
# Luego volver a loguear

También probá con el renderizador específico:

SDL_VIDEODRIVER=kmsdrm SDL_RENDER_DRIVER=opengles2 ./Quake3-UrT.x86_64

Opción 3: Recompilar ioQuake3 con soporte TTY nativo

Urban Terror 4.3 usa ioQuake3. Podés compilar tu propio binario ioQuake3 con soporte KMS y apuntarle los archivos .pk3 de UrT:

git clone https://github.com/ioquake/ioq3.git
cd ioq3
make USE_SDL=1 USE_OPENGL2=0

Luego ejecutarlo apuntando al directorio de UrT:

./build/release-linux-x86_64/ioquake3.x86_64 \
  +set fs_game q3ut4 \
  +set fs_basepath /home/esotericwarfare/urbanterror/UrbanTerror43

Opción 4: Usar un servidor X liviano en TTY (Xorg sin compositor)

La más compatible y simple. Instalás Xorg mínimo y lo arrancás desde TTY:

sudo apt install xorg xinit

Creás un ~/.xinitrc minimalista:

echo './Quake3-UrT.x86_64' > ~/.xinitrc
chmod +x ~/.xinitrc

Y desde la TTY:

startx

Xorg puede arrancar directo desde TTY usando KMS sin necesidad de un display manager.

Diagnóstico rápido recomendado

Primero probá esto para ver qué backends SDL2 están disponibles en tu sistema:

SDL_VIDEODRIVER=kmsdrm ./Quake3-UrT.x86_64 2>&1 | head -30
SDL_VIDEODRIVER=offscreen ./Quake3-UrT.x86_64 2>&1 | head -30

Y verificá qué versión de SDL2 tiene el binario enlazada:

ldd ./Quake3-UrT.x86_64 | grep sdl

Si el binario tiene SDL2 enlazada dinámicamente, podés reemplazarla con una versión compilada con soporte KMSDRM. Si está enlazada estáticamente, la Opción 3 (recompilar ioQ3) es tu mejor camino.
