No puedo jugar a UrbanTerror desde la TTY (abre el juego pero se cierra), asi que le pedi a Claude AI que me lo arregle.

Ya tengo todo lo que necesito. El problema está muy claro — te lo explico y después los pasos exactos para instalarlo y debuggearlo.

El problema raíz de Urban Terror en la TTY:

Urban Terror usa ioquake3, que a diferencia de GZDoom tiene un path de renderizado completamente distinto. GZDoom puede usar Vulkan o OpenGL con EGL directamente sobre KMS/DRM. ioquake3 (la versión que trae Urban Terror 4.3) fue compilada asumiendo que SDL encontrará un display X11 — cuando no lo hay, SDL intenta inicializar OpenGL sobre KMS/DRM, pero el código del engine llama a SDL_SetGammaRamp que en modo KMS/DRM sin X11 manda SIGSEGV (señal 11). La causa consistente del segfault es exactamente SDL_SetGammaRamp: Received signal 11, exiting, que es lo que te pasó.

La solución es un combo de tres cosas: instalar bien, deshabilitar el gamma ramp, y forzar el driver correcto. Acá van los pasos exactos:

# ── PASO 1: Instalar dependencias ──────────────────────────────────────────
sudo pacman -S lib32-libstdc++5 lib32-glibc lib32-sdl2 lib32-openal \
               lib32-libvorbis lib32-libjpeg-turbo wget unzip

# ── PASO 2: Descargar Urban Terror 4.3.4 ──────────────────────────────────
mkdir -p ~/games/urbanterror && cd ~/games/urbanterror

# Descarga directa del instalador oficial (Linux, ~800MB)
wget https://cdn.urbanterror.info/urt/43/releases/zips/UrbanTerror43_full.zip
unzip UrbanTerror43_full.zip
cd UrbanTerror43

# ── PASO 3: Verificar el binario que incluye ──────────────────────────────
ls -la ioUrbanTerror*
# Vas a ver: ioUrbanTerror.i386 y/o ioUrbanTerror.x86_64

# ── PASO 4: El fix del segfault — deshabilitar gamma ──────────────────────
# Editá el archivo de config ANTES de lanzar:
mkdir -p ~/.q3a/q3ut4
cat >> ~/.q3a/q3ut4/q3config.cfg << 'EOF'
seta r_ignorehwgamma "1"
seta r_fullscreen "0"
seta r_mode "-1"
seta r_customwidth "1280"
seta r_customheight "720"
EOF

# ── PASO 5: Intentar lanzar con SDL forzado a KMS/DRM ─────────────────────
# Primero asegurate de estar en el grupo video:
sudo usermod -aG video,input $USER
# (necesitás re-loguear o usar: newgrp video)

# Lanzar forzando el driver KMS/DRM de SDL (sin X11):
SDL_VIDEODRIVER=kmsdrm ./ioUrbanTerror.x86_64 +set r_ignorehwgamma 1

# Si ese segfaultea también, probá con offscreen (para diagnóstico):
SDL_VIDEODRIVER=offscreen ./ioUrbanTerror.x86_64 +set dedicated 0

# ── PASO 6: Capturar el error exacto ─────────────────────────────────────
# Corré esto para ver el backtrace completo:
SDL_VIDEODRIVER=kmsdrm ./ioUrbanTerror.x86_64 \
    +set r_ignorehwgamma 1 \
    +set com_logfile 1 \
    +set com_journal 0 \
    2>&1 | tee ~/urt_error.log

# El log queda en ~/.q3a/q3ut4/qconsole.log también
cat ~/.q3a/q3ut4/qconsole.log

Si sigue fallando — plan B con Quake3e-UrbanTerror:

Hay un fork moderno del engine llamado Quake3e-UrbanTerror que está basado en el último source dump de ioquake3 sin SDL, con todas las correcciones aplicadas — y reporta entre 10% y 200% más FPS que el engine original. Ese engine maneja KMS/DRM mucho mejor:

# Clonar y compilar Quake3e para Urban Terror
git clone https://github.com/rorgoroth/Quake3e-UrbanTerror.git
cd Quake3e-UrbanTerror

sudo pacman -S cmake vulkan-headers vulkan-icd-loader mesa

cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel $(nproc)

# Copiar el binario al directorio de UrT:
cp build/quake3e.ded ~/games/urbanterror/UrbanTerror43/

# Lanzar (sin SDL, usa Vulkan/DRM directo):
cd ~/games/urbanterror/UrbanTerror43
SDL_VIDEODRIVER=kmsdrm ./quake3e.ded +set sv_pure 0

Instalalo primero con los pasos 1-6, lanzalo, copiarme el contenido de ~/urt_error.log o de ~/.q3a/q3ut4/qconsole.log y con eso veo exactamente en qué línea revienta y si el fix del gamma ramp alcanza o necesitamos el engine alternativo.
