Backup entire site to Archive.org

May 29, 2026
Cli

Hay varias herramientas, pero hay una que es la más completa y oficial para esto. Te explico el panorama completo:

Contexto importante: dos cosas distintas

Archive.org tiene dos servicios separados:

  1. Wayback Machine — guarda snapshots de URLs públicas para que sean accesibles vía web.archive.org/web/...
  2. Archive.org como hosting — subís archivos directamente como ítems (como si fuera un repositorio)

Para tu caso (página personal legible con artículos), lo más útil es enviar tus URLs al Wayback Machine en masa, no subir archivos sueltos.

La herramienta recomendada: waybackpy

Es una CLI que interfaz directamente con las APIs públicas del Wayback Machine, incluyendo la Save API (Save Page Now), la Availability API y la CDX API.

Instalación en Arch Linux

pip install waybackpy --break-system-packages
# o con pipx (recomendado en Arch):
pipx install waybackpy

Uso básico para archivar una URL

waybackpy --save --url "https://tupagina.com/articulo-1" --user_agent "mi-bot/1.0"

Flujo completo para archivar toda tu página

Lo más limpio es combinarlo con tu sitemap.xml (si tenés uno) o generar una lista de URLs:

Opción A — Desde sitemap.xml

# Extraer URLs del sitemap y guardarlas en un archivo
curl https://tupagina.com/sitemap.xml | grep -oP '(?<=<loc>)[^<]+' > urls.txt

# Archivar cada URL con waybackpy
while IFS= read -r url; do
  echo "Archivando: $url"
    waybackpy --save --url "$url" --user_agent "mi-bot/1.0"
      sleep 5  # importante: no spammear el servidor
      done < urls.txt
      ```

      ### Opción B — Con `wayback-machine-archiver` (más robusto, requiere cuenta)

      A partir de la versión 3.0, esta herramienta requiere autenticación con la API SPN2 del Internet Archive. Las claves S3-style se obtienen desde la configuración de tu cuenta en `https://archive.org/account/s3.php`.

      ```bash
      pip install wayback-machine-archiver --break-system-packages

      # Crear archivo .env con tus claves
      echo "ACCESS_KEY=tu_access_key" > .env
      echo "SECRET_KEY=tu_secret_key" >> .env

      # Archivar desde sitemap
      archiver --sitemap https://tupagina.com/sitemap.xml
      ```

      ### Opción C — `wayback-sitemap-archive` (la más simple, vía npm)

      Se puede usar sin instalar con `npx wayback-sitemap-archive <URL_DEL_SITEMAP>`, o instalarlo globalmente con `npm install --global wayback-sitemap-archive`.

      ```bash
      npx wayback-sitemap-archive https://tupagina.com/sitemap.xml
      ```

      ---

      ## Si no tenés sitemap.xml

      Podés generarlo fácilmente. Si usás Hugo, Jekyll, o similar, ya lo generan automáticamente. Si es HTML estático:

      ```bash
      # Instalar wget y generar lista de URLs del sitio
      wget --spider --recursive --no-verbose --output-file=crawl.log https://tupagina.com
      grep -oP 'https?://tupagina\.com[^ ]+' crawl.log | sort -u > urls.txt
      ```

      ---

      ## Resumen de qué usar

      | Situación | Herramienta |
      |---|---|
      | Tenés sitemap y querés algo simple | `wayback-sitemap-archive` (npx) |
      | Querés control total desde bash | `waybackpy` en un loop |
      | Sitio grande, necesitás confirmación de éxito | `wayback-machine-archiver` (requiere cuenta IA) |

      El punto clave es que ninguna de estas "hostea" tu página en Archive.org — lo que hacen es decirle al Wayback Machine que vaya a *crawlear* tus URLs ya públicas. **Tu sitio tiene que estar en algún servidor** (GitHub Pages, Netlify, etc.) y las herramientas simplemente le piden a Archive.org que guarde copias accesibles.
