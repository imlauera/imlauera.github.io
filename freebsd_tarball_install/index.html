Freebsd tarball install

May 08, 2026
Freebsd

Vi tu guía de Arch. Es muy completa y práctica. Me gusta especialmente que documentás todo paso a paso con comandos concretos y explicaciones del “por qué” (GPT vs MBR, --removable para UEFI con fallas, etc.).

Ahora te armo una guía de instalación de FreeBSD desde tarball al estilo Arch — manteniendo tu formato (comandos directos, explicaciones mínimas pero útiles, y notas de troubleshooting). La estructura será: particionado → formateo → extracción → configuración → bootloader → chroot → servicios.

Instalación de FreeBSD desde tarball (estilo Arch Linux)

Paso 0: Preparar el entorno desde Arch Linux

loadkeys es
timedatectl set-ntp true

Necesitás herramientas para manipular particiones GPT y sistemas de archivos UFS/ZFS:

sudo pacman -S gptfdisk dosfstools
# Si querés ZFS (recomendado en FreeBSD):
sudo pacman -S zfs-utils
sudo modprobe zfs

Paso 1: Conectarte a WiFi (si es necesario)

iwctl
device list
station wlan0 scan
station wlan0 connect NOMBRE_RED
station wlan0 show
exit

Probá conexión:

ping -c 3 freebsd.org

Paso 2: Particionar el disco

¡ATENCIÓN! Esto destruye todos los datos de /dev/sdX. Verificá con lsblk antes.

lsblk
gdisk /dev/sdX

Dentro de gdisk:

o              # crear nueva tabla GPT
y              # confirmar

n              # nueva partición
1              # número
[Enter]        # primer sector por defecto
+512M          # último sector (EFI)
ef00           # tipo EFI System

n
2
[Enter]
+8G            # ajustá según tu RAM
8200           # tipo Linux swap (FreeBSD la usa igual)

n
3
[Enter]
[Enter]        # resto del disco
a503           # tipo FreeBSD UFS (o a504 para ZFS)

w              # escribir cambios
y              # confirmar

Paso 3: Formatear particiones

EFI:

mkfs.vfat -F 32 /dev/sdX1

Swap:

mkswap /dev/sdX2
swapon /dev/sdX2

Raíz — Opción A: UFS (más simple, similar a ext4):

newfs /dev/sdX3
# Nota: newfs es el mkfs.ufs de FreeBSD, pero desde Arch no lo tenés.
# Alternativa: usá la ISO de FreeBSD en modo live o descargá los tools.
# Por eso, si estás en Arch puro, ZFS es más fácil porque tenés zfs-utils.

Raíz — Opción B: ZFS (recomendado, nativo en FreeBSD):

zpool create -f -o altroot=/mnt/freebsd -O mountpoint=/ zroot /dev/sdX3
zfs create zroot/ROOT/default
zfs set mountpoint=/ zroot/ROOT/default
zpool set bootfs=zroot/ROOT/default zroot

Nota: Si usás UFS desde Arch Linux necesitás las herramientas ufsutils que no están en los repos oficiales. ZFS es más práctico desde Arch porque zfs-utils sí está disponible.

Paso 4: Descargar los tarballs de FreeBSD

cd /tmp
# Usá el mirror más cercano. Ejemplo para Argentina/Chile:
wget https://download.freebsd.org/releases/amd64/15.0-RELEASE/base.txz
wget https://download.freebsd.org/releases/amd64/15.0-RELEASE/kernel.txz
wget https://download.freebsd.org/releases/amd64/15.0-RELEASE/lib32.txz   # opcional, compat 32-bit

# Verificá integridad:
wget https://download.freebsd.org/releases/amd64/15.0-RELEASE/CHECKSUM.SHA512
sha512sum -c CHECKSUM.SHA512 --ignore-missing

Paso 5: Extraer los tarballs

Si usaste ZFS:

mkdir -p /mnt/freebsd
zfs set mountpoint=/mnt/freebsd zroot/ROOT/default
cd /mnt/freebsd
tar xvJf /tmp/base.txz
tar xvJf /tmp/kernel.txz
tar xvJf /tmp/lib32.txz   # si lo descargaste

Si usaste UFS (y lograste formatearla):

mkdir -p /mnt/freebsd
mount /dev/sdX3 /mnt/freebsd
cd /mnt/freebsd
tar xvJf /tmp/base.txz
tar xvJf /tmp/kernel.txz

Paso 6: Configurar el bootloader EFI

FreeBSD usa su propio loader.efi, no GRUB. Copialo a la partición EFI:

mkdir -p /mnt/freebsd/boot/efi
mount /dev/sdX1 /mnt/freebsd/boot/efi
mkdir -p /mnt/freebsd/boot/efi/EFI/BOOT
cp /mnt/freebsd/boot/loader.efi /mnt/freebsd/boot/efi/EFI/BOOT/BOOTX64.efi

Nota: Si tu UEFI es problemático (como los que borran entradas EFI), copiá también a la ruta fallback:

cp /mnt/freebsd/boot/loader.efi /mnt/freebsd/boot/efi/EFI/BOOT/BOOTX64.efi

FreeBSD no usa efibootmgr, su loader se autoinstala en la partición EFI.

Paso 7: Preparar el chroot y entrar

mount --bind /dev /mnt/freebsd/dev
mount --bind /proc /mnt/freebsd/proc
mount --bind /sys /mnt/freebsd/sys
cp /etc/resolv.conf /mnt/freebsd/etc/
chroot /mnt/freebsd /bin/sh

Paso 8: Configurar el sistema dentro del chroot

8.1 fstab (swap y puntos de montaje):

cat > /etc/fstab << 'EOF'
/dev/gpt/swap    none    swap    sw    0    0
EOF

En FreeBSD los discos GPT se nombran /dev/gpt/label o /dev/ada0p1. Si no usaste labels, podés referirte por UUID o dispositivo. Para instalación simple, usá /dev/ada0p2 (ajustá según tu hardware real).

8.2 ZFS (si aplica):

echo 'zfs_load="YES"' >> /boot/loader.conf
echo 'vfs.root.mountfrom="zfs:zroot/ROOT/default"' >> /boot/loader.conf
echo 'zfs_enable="YES"' >> /etc/rc.conf

8.3 Hostname y red:

echo 'hostname="freebsd"' >> /etc/rc.conf
echo 'ifconfig_DEFAULT="DHCP"' >> /etc/rc.conf   # Ajustá si tu interfaz no es em0/re0/etc
echo 'sshd_enable="YES"' >> /etc/rc.conf
echo 'dumpdev="NO"' >> /etc/rc.conf              # desactiva crash dumps si no querés partición swap dedicada

8.4 Zona horaria:

tzsetup
# Seleccioná tu zona interactivamente (America/Buenos_Aires)

8.5 Locale:

echo 'LANG=en_US.UTF-8' > /etc/login.conf
# FreeBSD maneja locales diferente a Arch. Para consola:
echo 'keymap="es.kbd"' >> /etc/rc.conf

8.6 Contraseña root:

passwd

8.7 Crear usuario:

adduser
# Seguí las preguntas. Añadilo al grupo 'wheel' si querés usar su/sudo después.

Paso 9: Instalar sudo y configurar (opcional pero recomendado)

pkg install sudo
# O si pkg no está configurado todavía, esperá a bootear y hacerlo desde el sistema vivo.

Para habilitar sudo para el grupo wheel:

visudo
# Descomentá: %wheel ALL=(ALL:ALL) ALL

Paso 10: Salir del chroot y desmontar

exit                              # salir del chroot
umount /mnt/freebsd/dev
umount /mnt/freebsd/proc
umount /mnt/freebsd/sys
umount /mnt/freebsd/boot/efi
zpool export zroot               # si usaste ZFS
umount /mnt/freebsd              # si usaste UFS
swapoff /dev/sdX2

Paso 11: Rebootear

reboot

Desconectá cualquier medio de instalación. El sistema debería bootear directo al loader de FreeBSD.

Paso 12: Post-instalación (ya dentro de FreeBSD)

Actualizar el sistema:

freebsd-update fetch
freebsd-update install

Instalar el stack gráfico (equivalente a tu xorg + sway en Arch):

pkg install xorg-minimal sway foot wmenu waybar
# o si querés algo más completo:
pkg install xorg sway i3status dmenu st

Audio (equivalente a pipewire):

pkg install pulseaudio
# FreeBSD usa OSS nativamente, pero pulseaudio funciona como capa de compatibilidad.

Gestor de paquetes (equivalente a pacman):

pkg install pkg
pkg update
pkg search nombre_paquete
pkg install nombre_paquete

Notas de troubleshooting

Problema Solución
No bootea, UEFI no encuentra el loader Asegurate que BOOTX64.efi esté en /EFI/BOOT/. Algunos firmwares necesitan el archivo en la raíz de la EFI también.
ZFS no importa al bootear Desde el loader de FreeBSD (cuando aparece OK en el boot), presioná 3 para entrar al loader prompt y ejecutá: set vfs.root.mountfrom="zfs:zroot/ROOT/default" luego boot
No hay red después de bootear Verificá con ifconfig. FreeBSD nombra interfaces em0, re0, igb0, etc. Ajustá /etc/rc.conf con el nombre correcto.
pkg no funciona (no encuentra repos) Ejecutá pkg bootstrap -f o configurá /etc/pkg/FreeBSD.conf
Querés compilar desde fuentes como Gentoo Descargá src.txz, extraelo en /usr/src, y usá make buildworld buildkernel installkernel installworld

Diferencias clave con tu guía de Arch

Arch Linux FreeBSD
pacstrap tar xvJf base.txz
genfstab Editás /etc/fstab manualmente
grub-install Copiás loader.efi a la partición EFI
arch-chroot chroot /mnt /bin/sh (manual)
systemctl enable echo 'service_enable="YES"' >> /etc/rc.conf
pacman -S pkg install
mkinitcpio No existe. FreeBSD compila el kernel con los módulos necesarios.
ext4 UFS o ZFS (ZFS es nativo y recomendado)

¿Querés que profundice en algún paso? Por ejemplo, la instalación con UFS desde Arch es más complicada porque faltan las ufsutils en los repos.
