2. Algoritmo de descifrado

1. atob(blob base64)                        → string plano (141KB → ~105KB)
2. split en 4 chunks de tamaño ceil(n/4)
3. Por cada chunk:
   a. Eliminar el 4° carácter (índice 3)
   b. atob(chunk modificado)               → string parcial
   c. Reordenar con permutación [2, 0, 3, 1]
4. Join de los 4 chunks descifrados
5. atob(resultado)                         → JSON final

Implementación en Python

import base64, math, json

def decode_econfig(econfig_str):
    decoded = base64.b64decode(econfig_str).decode('latin-1')
    n = len(decoded)
    chunk_size = math.ceil(n / 4)
    chunks = []
    pos = 0
    for i in range(4):
        chunks.append(decoded[pos:pos+chunk_size])
        pos += chunk_size

    perm = [2, 0, 3, 1]
    result_arr = [None] * 4
    for i in range(4):
        chunk = chunks[i]
        modified = chunk[:3] + chunk[4:]  # remove index 3
        decoded_chunk = base64.b64decode(modified).decode('latin-1')
        result_arr[perm[i]] = decoded_chunk

    joined = ''.join(result_arr)
    final_bytes = base64.b64decode(joined)
    return json.loads(final_bytes.decode('utf-8'))

  2. El HMAC es válido
  3. El timestamp no expiró

9. Comparativa: proveseat.net vs ninguno.net

Aspecto proveseat.net ninguno.net (f1box)
Player Clappr + hls.js + p2p-media-loader Oculto en iframe
Protocolo HLS (.m3u8) con P2P No determinado
DRM ClearKey (en algunos streams) No determinado
P2P Sí (WebRTC, tracker: hlspatch.net) No detectado
Anti-forense _econfig ofuscado, sandbox_block, devtools_block CSRF HMAC + IP binding
Ofuscación base64 triple capa + permutación de chunks No aplica
CDN del stream nginx en 194.48.200.24 (Altrosky/Seychelles) Desconocido
Infraestructura Propia (IP fija, Let’s Encrypt) DNS rotatorio/oculto
Frontends deportelibree.com, cartelive.club, zonatv.store f1box.co, embedsports.me
Facilidad de análisis Alta (_econfig descifrable) Baja (todo en iframe protegido)

Similitudes

  1. Modelo de negocio: Ambos proveen infraestructura a frontends con publicidad
  2. Protección multi-capa: Iframes + tokens + validación
  3. Zero hosting propio: Los frontend no alojan nada
  4. Tracking: Ambos trackean visitas para medir audiencia

Diferencias clave

  1. Nivel de ofuscación: proveseat.net expone su config (aunque ofuscada); ninguno.net la oculta completamente en un iframe con CSRF
  2. Player: proveseat.net usa Clappr + P2P; ninguno.net es un backend cerrado
  3. DNS: proveseat.net tiene IP fija (194.48.200.24); ninguno.net no resuelve
  4. Antiforense: ninguno.net es más agresivo (CSRF + IP binding en vez de ofuscación client-side)

10. Métodos reales para obtener streams originales

Estos sitios no “encuentran las claves” — tienen acceso al contenido original mediante estos métodos:

Método 1: Account Farming + DevTools Automation (el más común)

Alguien en el backend paga una suscripción real a DAZN/Sky/ESPN y automatiza la extracción:

1. Abre DAZN en Chrome con cuenta legítima
2. F12 → Network → filtrar ".mpd" o ".m3u8"
3. Copia la URL del CDN (akamaized.net) + las ClearKey keys
4. Automatiza con Puppeteer/Playwright para renovar URLs cada 24h
5. Las mete detrás de su propio proxy con nginx + token signing

Una suscripción de $50-100/mes genera URLs que se revenden a miles de usuarios.

Método 2: Rooted Android TV + MITM Proxy

1. TV Box Android con DAZN app + suscripción real
2. Instalan mitmproxy/Charles con certificado root
3. Capturan todas las peticiones .mpd/.m3u8 y license keys
4. Extraen el device ID y lo clonan para generar sesiones fresh

Las apps de Smart TV suelen tener protecciones más débiles que el browser.

Método 3: Reverse Engineering de la App

  1. Decompilan la app de DAZN/Sky con jadx o apktool
  2. Encuentran los endpoints de API que generan los stream URLs
  3. Replican el handshake de autenticación
  4. Si usa Widevine L3 (software), extraen las keys del memory dump

Método 4: CDN Token Leak / Insider Access

Un empleado del CDN (Akamai, CloudFront) o del broadcaster filtra la clave de firma de tokens. Es raro pero ocurre.

Método 5: Leeching de Sesiones Compartidas

Algunos servicios tienen “family sharing” o “multi-screen”. Los piratas compran una cuenta “family” y generan múltiples streams simultáneos desde diferentes IPs.

11. Modelo de negocio completo

Backend (proveseat.net / ninguno.net)
├── Paga $50-100/mes por suscripción real a DAZN/Sky
├── Extrae stream URLs + DRM keys (DevTools/MITM automation)
├── Los envuelve en su propio proxy/CDN con tokens firmados
└── Vende acceso a 20-50 frontends ($100-500/mes c/u)

Cada Frontend (f1box.co, deportelibree.com, etc.)
├── Paga al backend por el embed/iframe
├── Llena la página de publicidad pop-up
│   ├── Popads / PopCash / AdMaven
│   ├── Redirects a casinos/gambling
│   └── Botón "Watch Live" falso = ad redirect
├── Gana ~$1000-5000/mes en publicidad
└── Si los tumban, abren otro dominio en horas

Usuario final
├── Ve el stream "gratis"
├── Soportando 3-5 ventanas pop-up
├── 1 botón falso de publicidad
└── Probablemente malware en algún lado

12. Netflix vs DAZN/Sky: Diferencia de DRM

Aspecto DAZN/Sky/ESPN Netflix/Disney+/HBO
DRM ClearKey Widevine L1/L3 o FairPlay
Dónde está la key En el JS del player (visible) Dentro del CDM (módulo sellado)
Extracción F12 → Network → copiar key Requiere romper el CDM
Herramientas DevTools pywidevine, CDM extract

Cómo piratean Netflix

  1. Widevine L3 extraction — En PC, Widevine corre en software (L3). Herramientas como pywidevine o wks-keys extraen la clave privada del CDM de Chrome. Después, esa clave se usa para descifrar licenses de cualquier contenido.

  2. Rooted Android — En algunos devices Android, el CDM corre en L3 (software) y se puede extraer el private_key.pem y client_id.bin del almacenamiento.

  3. Cached CDM — Una vez extraído un CDM funcionando, se comparte en grupos privados. Todos los que tienen ese CDM pueden descifrar Netflix.

  4. Automatización total — Tools como netflix-1080p o streamlink usan Chrome headless + CDM extraído → loguean con cuentas compartidas → descargan el stream descifrado.

Por qué Netflix no usa ClearKey

ClearKey es básicamente “security theater” — la key está a la vista en el JS. Netflix paga por Widevine porque necesitan proteger realmente su contenido. Los canales de TV en vivo (DAZN, ESPN) a veces usan ClearKey por:

  • Costo (Widevine cuesta licencias)
  • Compatibilidad (funciona en cualquier browser sin plugins)
  • El contenido es “en vivo” — menos valioso para piratear a largo plazo

3.2. CSRF + IP Binding

El token CSRF es un HMAC que incluye:

base64(IP) + "." + base64(timestamp) + "." + base64(hmac_secreto)

Ejemplo real:

MTgxLjk4LjI0Mi41NA            ← base64("181.98.242.54")
.MTc3OTU5MzU0Ng               ← base64(timestamp)
.NWIyOGUzMGM3ZjljNDk1MGViNDAzZDQ5MWM2NjIyMjk2YzQyOWZjOTlkZmExMGU5NDcwNjZjMmRiYTY2NDlhZg

El backend verifica que:

  1. La IP del request coincida con csrf_ip
  2. El HMAC sea válido
  3. El timestamp no haya expirado

3.3. DNS

  • sts.ninguno.net → no resuelve
  • embedsports.me → no resuelve

Esto sugiere que usan DNS dinámico o cambian de IPs frecuentemente.

7. Métodos Reales para Obtener Streams Originales

Estos sitios no “encuentran las claves” — tienen acceso al contenido original mediante estos métodos:

Método 1: Account Farming + DevTools Automation (el más común)

Alguien en el backend paga una suscripción real a DAZN/Sky/ESPN y automatiza la extracción:

1. Abre DAZN en Chrome con cuenta legítima
2. F12 → Network → filtrar ".mpd" o ".m3u8"
3. Copia la URL del CDN (akamaized.net) + las ClearKey keys
4. Automatiza con Puppeteer/Playwright para renovar URLs cada 24h
5. Las mete detrás de su propio proxy con nginx + token signing

Una suscripción de $50-100/mes genera URLs que se revenden a miles de usuarios.

Método 2: Rooted Android TV + MITM Proxy

1. TV Box Android con DAZN app + suscripción real
2. Instalan mitmproxy/Charles con certificado root
3. Capturan todas las peticiones .mpd/.m3u8 y license keys
4. Extraen el device ID y lo clonan para generar sesiones fresh

Las apps de Smart TV suelen tener protecciones más débiles que el browser.

Método 3: Reverse Engineering de la App

  1. Decompilan la app de DAZN/Sky con jadx o apktool
  2. Encuentran los endpoints de API que generan los stream URLs
  3. Replican el handshake de autenticación
  4. Si usa Widevine L3 (software), extraen las keys del memory dump

Método 4: CDN Token Leak / Insider Access

Un empleado del CDN (Akamai, CloudFront) o del broadcaster filtra la clave de firma de tokens. Es raro pero ocurre.

Método 5: Leeching de Sesiones Compartidas

Algunos servicios tienen “family sharing” o “multi-screen”. Los pirates compran una cuenta “family” y generan múltiples streams simultáneos desde diferentes IPs.

#!/usr/bin/env python3
"""
Decodificador _econfig de proveseat.net
Algoritmo: base64 -> split 4 chunks -> remove 4th char of each ->
atob each -> reorder [2,0,3,1] -> join -> atob -> JSON
"""
import base64, math, json, sys

def decode_econfig(econfig_str):
    decoded = base64.b64decode(econfig_str).decode('latin-1')
    n = len(decoded)
    chunk_size = math.ceil(n / 4)
    chunks = []
    pos = 0
    for i in range(4):
        chunks.append(decoded[pos:pos+chunk_size])
        pos += chunk_size

    perm = [2, 0, 3, 1]
    result_arr = [None] * 4
    for i in range(4):
        chunk = chunks[i]
        modified = chunk[:3] + chunk[4:]
        decoded_chunk = base64.b64decode(modified).decode('latin-1')
        result_arr[perm[i]] = decoded_chunk

    joined = ''.join(result_arr)
    final_bytes = base64.b64decode(joined)
    return json.loads(final_bytes.decode('utf-8'))

if __name__ == '__main__':
    if len(sys.argv) > 1:
        with open(sys.argv[1]) as f:
            data = f.read().strip()
    else:
        data = sys.stdin.read().strip()
    result = decode_econfig(data)
    print(json.dumps(result, indent=2, ensure_ascii=False))

9. P2P + WebRTC: Cómo funciona p2p-media-loader-hlsjs

Concepto

Usa WebRTC (el mismo protocolo que Zoom, Google Meet, Discord) para que los espectadores se compartan fragmentos del video entre sí, reduciendo drásticamente el ancho de banda que el servidor principal debe servir.

Flujo del P2P

Servidor HLS original (stream_url)
  → Sirve fragmentos .ts a los primeros N usuarios
    → Esos usuarios comparten fragmentos (que ya descargaron)
      a nuevos usuarios via WebRTC DataChannels
        → Un tracker BitTorrent coordina quién tiene qué fragmento

Componentes

Componente Rol
swarmId Identifica un “grupo” de peers. Todos viendo el mismo canal pertenecen al mismo swarm
Tracker (announceTrackers) Servidor WebSocket que dice a cada peer qué otros peers están en el swarm
WebRTC DataChannels Conexión directa peer-to-peer para transferir fragmentos .ts
HlsJsP2PEngine Capa que intercepta hls.js y redirige peticiones al P2P cuando es posible

Código (extraído de stream.js)

import { Core } from 'p2p-media-loader-core';
import { HlsJsP2PEngine } from 'p2p-media-loader-hlsjs';

const engine = new HlsJsP2PEngine({
    core: {
        swarmId: config.swarmId,              // ej: "dazn-f1-es"
        announceTrackers: [config.p2p_tracker]  // tracker WebSocket
    }
});

// Se inyecta en la config de hls.js dentro de Clappr
playerConfig.playback.hlsjsConfig = {
    ...engine.getConfigForHlsJs()
};

// Clappr usa hls.js internamente con el P2P engine
new Clappr.Player({ source: streamUrl, ...playerConfig });

Por qué usan P2P

  1. Reducción de costos. El servidor paga por ancho de banda. Si el 70-80% del tráfico se distribuye entre usuarios via P2P, el costo se reduce drásticamente.
  2. Escalabilidad. Un evento con 100k viewers no mata el servidor porque los viewers se sirven entre sí.
  3. Resiliencia. Si el servidor cae, los peers que ya tienen fragmentos pueden seguir compartiéndolos.
  4. Ideal para contenido en vivo. Todos ven los mismos fragmentos al mismo tiempo → alta tasa de aciertos P2P.

Similitudes

Tecnología Uso similar
PeerTube P2P para video bajo demanda (ActivityPub)
StreamMux / Aura CDN P2P para streaming comercial
BTFS BitTorrent File System
WebTorrent BitTorrent en el navegador via WebRTC
p2p-media-loader Libería open-source usada aquí

Implicancia forense

El P2P hace que desmantelar el sitio sea más difícil porque parte de la infraestructura son los propios usuarios. No alcanza con tumbar el servidor:

  • Cada usuario con la página abierta es un “seed” que redistribuye el stream
  • El tracker puede ser cualquiera (incluso el mismo proveseat.net)
  • Los DataChannels son cifrados (DTLS) — no se puede inspeccionar qué fragmentos comparten
  • El swarmId puede rotar para evadir bloqueos

Para desmantelamiento real, habría que:

  1. Tumbar el tracker (único punto centralizado)
  2. Tumbar la fuente del stream original (stream_url)
  3. O presionar al proveedor del CDN donde se hostea el stream legítimo (Akamai, Cloudfront)

Lo que no funciona: bloquear dominios (tienen 134+), DMCA a Cloudflare (datos falsos), o cerrar frontends (migran en horas).

6. Anexo: Decryptado Completo de window._econfig

Logrado el 2026-05-24. Herramienta: decode_econfig.py

El _econfig de proveseat.net/embed/2twhx (141KB base64) se descifró exitosamente. El algoritmo resultó ser:

Algoritmo de descifrado

1. atob(blob base64)                        → string plano
2. split en 4 chunks de tamaño ceil(n/4)
3. Por cada chunk:
   a. Eliminar el 4° carácter (índice 3)
   b. atob(chunk modificado)               → string parcial
   c. Reordenar con permutación [2, 0, 3, 1]
4. Join de los 4 chunks descifrados
5. atob(resultado)                         → JSON final

def decode_econfig(econfig_str):
    decoded = base64.b64decode(econfig_str).decode('latin-1')
    n = len(decoded)
    chunk_size = math.ceil(n / 4)
    chunks = [decoded[i:i+chunk_size] for i in range(0, n, chunk_size)]
    perm = [2, 0, 3, 1]
    result_arr = [None] * 4
    for i in range(4):
        mod = chunks[i][:3] + chunks[i][4:]  # remove index 3
        result_arr[perm[i]] = base64.b64decode(mod).decode('latin-1')
    return json.loads(base64.b64decode(''.join(result_arr)))
